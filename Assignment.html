This semester you have to program a simple game (othello) in python3 (see here how to install python)

The following list under Assignment tasks gives you instructions to create step by step (starting by 1 you should complete each task one by one) a playable game. We are giving you a template which could be used and gives you an easier start.

Assignment tasks (10 points for each first level list item)

  1. Let’s start with a stone
    1. Look at the code provided and make yourself familiar. 
    2. Create a function to set a stone to one field.
    3. Print the updated board to the console.
  2. It’s your turn - keyboard time
    1. At this point of the assignment we want to be able to play the game with our keyboard. The game should be played one player after one, so starting by asking where to put the first black stone, print the whole field to the console accordingly, asking where to put the first white, and so on. 
    2. Coordinates are defined by row (A-H)[only upper case] and column (0-7), e.g. A0. 
    3. PROMPT_PLAYER_1 and PROMPT_PLAYER_2 are shown on a rotating basis depending on the current player. (see screenshots at the end of the page) 
  3. Error detection and handling - What can go wrong?:
    1. Detect errors like when a player tries to occupy an already full field. What if a cat runs across your keyboard, does your game crash? Make sure the player order is fair and nobody takes two turns. Don't allow drawing outside the board.
    2. ERROR_INVALID_INPUT must be printed if the input is not as expected
    3. ERROR_OCCUPIED must be printed if the field is already full.
  4. Color changing times
    1. Have a look at https://de.wikipedia.org/wiki/Othello_(Spiel) and make yourself familiar to the game rules.
    2. If your placed stone enclose an opposing stone with one of your stones, the the opposing stone is now yours. 
    3. Check after every time a stone is played if the new placed stone is able to change the color of an opposing stone.
      1. Stones that just changed its color are not included in the enclosure rule. (else every possible stone would be turned)
  5. Rules… Rules… Rules…
    1. The start of the game is defined by four stones in the center. Let’s start every game like that.
    2. In Othello you are only allowed to put your stone in a field next to an opposing stone (vertical, horizontal, diagonal), if you are able to change the color from minimum one opposing stone. 
      1. Check before the stone is placed, if the selected field is currently allowed. 
        1. If so, draw the stone and check for color changing events.
        2. If not, make sure the user knows it is not allowed to put the stone there and the player is able to select another field.
      2. Since sometimes the player is not able to play a valid turn, enable the player to “skip” the turn.
      3. If the player want's to play an illegal field in terms of the rules, print ERROR_NOT_ALLOWED  
      4. If any error occurs, the previous prompt should be asked again. (In the whole game)
  6. Winning, the Art of loosing at Loosing
    1. If both player pass the turn (both are not able to play a valid move) or every field is set, count the stones. The player with more stones on the board wins, if they are equal it’s a draw. Make sure the player is informed the game is over and who has won (or if it’s a draw [both players have kinda lost])
    2. Statistics and won message
      1. Print at the end of the game the statistics of the game in two lines (see figure)
      2. After the two statistic lines print the game result (WON_PLAYER_1, WON_PLAYER_2 or WON_DRAW)
    3. Exit codes
      1. If player 1 has won, exit the program with exit code 1
      2. If player 2 has won, exit the program with exit code 2
      3. If it's a draw,  exit the program with exit code 3
      4. When the player enters 'quit' the program should stop without any other print and return exit code 0. No statistics should be displayed!
  7. Oh, we have a computer
    1. Since we play the game on a computer, let’s program a computer opponent.
    2. Start with a simple strategy, computer wants to put the stone to a random field that is allowed.
    3. Print PROMPT_HUMAN_AI at the beginning. If a string equal to INPUT_HUMAN is given, 2 humans are playing, if a string equal to INPUT_COMPUTER is given, player 1 is human, player 2 is the computer. 
      1. If the input is not equal to ether INPUT_HUMAN or INPUT_COMPUTER,  print ERROR_INVALID_INPUT and print the prompt PROMPT_HUMAN_AI again.
    4. Print instead of PROMPT_PLAYER_2 ; PROMPT_PLAYER_AI and the selected coordinates of the ai. (see screenshots at the end of the page) 
  8. Aaaaand mankind is lost, the computer takes over the world…
    1. Analyse every field possible playing and rate how many stones are flipped in this move. The computer plays the field which flipps the most stones.
    2. If you program an ever better strategy, bonus points will be rewarded.
